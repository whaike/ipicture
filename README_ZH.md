# 家庭相册整理
扫描指定文件路径下的所有图片和视频进行去重,并将剩余有效信息记录至`ipicture.db`文件

## 功能与规则
### 去重
对于各种原因产生的重复图片或视频只保留唯一
### 归类
保留原有的分类文件夹,不对原文件或文件夹做修改,如果设置`delDuplicate`为`true`,则会从中删除重复的部分

## 使用
需要先安装 [exiftool](https://www.sno.phy.queensu.ca/~phil/exiftool/)

程序启动后会扫描指定目录下的所有文件,并在当前目录下生成`ipicture.db`文件,其中记录了去重后的可用的图片和视频文件的有效物理路径及相关信息.
程序最多只删除重复的文件,不对现有其他数据做任何修改。

### 编译
`./build.sh xxx` windows / arm64 / linux
### 运行
`./bin/arm64/ipicture -path=path to photos root directory`




## 一些特征
首先，相册有很多的分类，但是同一个设备拍摄出来的相片其命名方式大体差不多，不过这种相似性针对的是同种文件夹，比如水印相机拍的相似，截图的相似，原相机拍的相似等等。

其次，命名相似的可能包含不同种类的图片或者视频，其创建日期和拍摄日期可能相同，他们应该属于同一类。

## 程序逻辑
首先，我们认为用户的第一问题就是【重复+分类】的问题，也是最重要追直接的问题。

用户有很多图片，各种手机、相机游玩或随手拍的甚至截图的图片，备份到同一个或者多个大的目录下，希望的第一件事一定是解决重复以及分类的问题。

这里重复很好理解，不论遇到多少重复，只保留其中一个就行了。

分类这个动作存在个人的差异，有的希望按时间分，有的希望按地点分，有的希望按人物、风景分等等。但针对不同的人满足不同的心理预期是很难的，在开始的时候，我们只能选择一个大众普便能接受的处理方式，也是回归图片整理最原始、最容易接受的方式，**按时间轴**。


### 扫描入库
1、扫描目录，将遇到的每一个图片或者视频计算md5后，将路径和信息入库

2、如果入库前发现md5重复，则不再入库，如果路径相同则跳过，路径不同则删除当前扫描到的数据
